## Lab 3: Платформа історичних подій з React та маршрутизацією

### Опис завдання
У ході виконання лабораторної роботи №3 було створено односторінковий веб-додаток (SPA) з використанням фреймворку React та бібліотеки для маршрутизації React Router. Платформа призначена для вивчення історичних подій з інтерактивним тестуванням користувачів.

### Завдання 1 — Проект створено за допомогою Vite
Додаток запускається командою `npm run dev` та доступний за адресою http://localhost:5174.
- ✅ Налаштований moderne build tool Vite для оптимізованої розробки
- ✅ Встановлені необхідні залежності (React 19, React Router 7.13.1)
- ✅ Структурована архітектура проекту з папками src/, public/

### Завдання 2 — Реалізовано компонентну архітектуру: створено 6 перешляхування компонентів
**Основні компоненти:**
- **Navigation** - навігаційна панель для переміщення між сторінками (Home, Lessons, Gallery, Progress)
- **LessonCard** - компонент для відображення окремої історичної подій
- **GalleryItem** - компонент для展示галереї подій
- **ProgressBar** - компонент для відображення прогресу користувача
- **Timeline** - інтерактивна шкала часу для 7 історичних подій
- **Modal** - компонент для відображення інформаційних вікон

Кожен компонент використовує JSX-синтаксис, має окремий CSS-файл для стилізації та приймає дані через props.

### Завдання 3 — Керування станом реалізовано за допомогою useState
App застосовує pattern «lifted state» для масиву уроків та результатів, який передається через props у сторінки.

**Lessons та Progress** - забезпечують синхронізацією даних між ними. Додатково стан використовується для управління мобільним меню в Navbar та фільтрацією галереї в Gallery.

### Завдання 4 — Маршрутизацію реалізовано за допомогою бібліотеки react-router-dom
Визначено 4 маршрути (/,/lessons, /gallery, /progress) та Fallback-маршрут. Навігація реалізована за допомогою компонента NavLink для переходу між сторінками з підсвічуванням активного посилання. Вмист сторінки динамічно оновляється без перезавантаження завдань компонентами Routes та Route.

### Теxнологічний стек
- **Frontend:** React 19, React Router DOM 7.13.1, Vite 8.0.0, CSS3
- **Управління даними:** historicalData.js (7 подій, 5 тестових запитань)
- **Стилізація:** Градієнти (#667eea → #764ba2), адаптивний дизайн

### Результати та висновки
✅ Успішно реалізована повнофункціональна SPA з компонентною архітектурою, маршрутизацією та керуванням станом. Платформа демонструє глибоке розуміння React та сучасних підходів до фронтенд-розробки.

## Lab 5: Full-Stack розробка - Express Backend + Firestore + API

### Опис завдання
У ході виконання лабораторної роботи №5 було розширено проект Lab 4 шляхом добавлення серверної частини Node.js/Express та облачної бази даних Firestore для збереження результатів тестів користувачів. Додаток розгорнутий на трьох платформах: Vercel (фронтенд), Render.com (бекенд), Firebase (БД та автентифікація).

### Завдання 1 — Налаштування серверної частини Node.js/Express
- ✅ Інітіалізований новий Express проект в папці `/server`
- ✅ Встановлені залежності: express, cors, firebase-admin, dotenv
- ✅ Налаштований сервер для запуску на порту 3000 (локально) та Render.com (продакшн)
- ✅ Реалізовано обробка помилок та graceful shutdown

### Завдання 2 — Облачна база даних Firestore та API endpoints

**Колекція `testResults` містить документи з полями:**
- `userId` - унікальний ідентифікатор користувача
- `email` - адреса користувача
- `score` - набрані бали в тесті
- `maxScore` - максимальні бали (100)
- `testName` - назва тесту ("Historical Events Quiz")
- `timestamp` - серверна мітка часу

**Реалізовані маршрути:**
- `GET /api/test-results/:userId` - отримання результатів користувача й обчисленням середньої оцінки
- `POST /api/test-results` - збереження нових результатів тестів
- `GET /health` - перевірка доступності сервера

### Завдання 3 — Frontend інтеграція з API

**Test.jsx оновлена для збереження результатів:**
- Додана функція `saveResultsToServer()` для POST запиту на API
- Після перевірки відповідей результат автоматично надсилається на сервер
- Виведення статусу ("Збереження...", "✓ Результат збережено")
- Error handling з інформуванням користувача

**ProgressChart.jsx оновлена для отримання даних:**
- Завантаження результатів користувача з сервера при монтуванні компонента
- Обчислення та відображення середньої оцінки з усіх спроб
- Показання кількості пройдених тестів
- Кольорова шкала для оцінок (зелена → червона)

### Вирішені виклики

| Проблема | Причина | Рішення |
|----------|---------|---------|
| Firebase Private Key Error | Неправильна передача ключа через env | Base64 кодування service account |
| CORS Errors | Бекенд не дозволяв фронтенду | Додання CORS middleware |
| Firestore Blocked | Security Rules були `if false` | Зміна на `if true` |
| Diverged Git Branches | Конфліктуючі коміти | `git pull --rebase` + reset |
